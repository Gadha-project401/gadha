(async, inner) getUser(remoteUser) → {object}

After a user object has been created, we check if the user is already signed up, if yes we give him a user sign in token, if not we save that use to the database and give him a token after that.
Parameters:
Name Type Description
remoteUser object After we get the user information, we want to save it in our database.
Properties:
Name Type Description
username string We combine the first name and the last name without spaces to create our username.
fullName string This is the full name of the use that was provided to us by linkedin.
gender string This is the gender of the user, linkedin doesn't provide it so we store it as unspecified until the user wants to change his data.
role string This is the user's role, it's always user and not admin.
birthday string This is the users birthday, linkedin doesnt share this information so we use a filler until the user changes it himself.
